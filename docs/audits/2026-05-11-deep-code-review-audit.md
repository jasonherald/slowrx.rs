## Clean bills of health (verified — not findings)

- All 11 `ModeSpec` timing constants match slowrx C byte-for-byte.
- No `unsafe` anywhere (`#![deny(unsafe_code)]` holds).
- No `unwrap` / `expect` / `panic!` in non-test library code (one defensive `expect` in `SstvImage::new`).
- `#[non_exhaustive]` applied consistently to public enums.
- The single-`round()` pixel-time formula is faithfully replicated everywhere it appears.
- YCbCr→RGB plus `clip()` rounding matches `common.c`.
- The SDX `+1` Hann-window bump tracks `video.c:367`.
- The chroma-dup last-row guard is present (R36/R24).
- The state machine cannot infinite-loop.
- `decode_one_channel_into`'s f64→i64 sample-index casts degrade to silence (not panic) on overflow.

| **A1** | High | Stale `VisDetector` left in place on unknown-VIS-code path | `decoder.rs:284-289` | The unknown-code branch does `let _ = self.vis.take_residual_buffer();` but never replaces `self.vis` — violating `take_residual_buffer`'s documented `#40` contract (a spent detector corrupts `end_sample` / pattern-window on the next detection). | **A2** | High | Multi-image-in-one-`process()`-call only decodes the first image | `decoder.rs:343-347`; manifests in `examples/decode_wav.rs:69` | After a decode it re-seeds the VIS detector then `break`s instead of `continue`-ing, so a 2nd/3rd VIS already in `trailing` isn't surfaced until another `process()` call. | **A3** | Medium | VIS detector returns unknown-but-parity-passing codes; slowrx keeps searching the 9 `(i,j)` alignments | `vis.rs:341-396` | slowrx `vis.c:121-123` rejects an unknown code *inside* the alignment loop and can still find a known code at a different `(i,j)`. Either pass an `is_known` predicate into `match_vis_pattern` (faithful) or document as an intentional simplification (the caller's retry loop covers most of it). Distinct from the documented "round-2 #5" deviation (that's about parity *failure*). | small |
| **A4** | Medium | Per-channel reset of `snr_db` / `prev_win_idx` diverges from slowrx C's image-wide stickiness — undocumented | `mode_pd.rs:444-457` | slowrx declares `SNR=0` / `WinIdx=0` *once per image*; `decode_one_channel_into` re-cold-starts them every channel (~2000×/image for PD, 3×/line otherwise). | **A6** | Low | `find_sync`'s 8-tap edge-find loop runs one extra window vs slowrx C | `sync.rs:340` | `x_acc.windows(8).enumerate()` → 693 windows (`0..=692`); slowrx C is `for (x=0; x<700-8; x++)` → 692 (`0..=691`), and `xAcc[699]` is never read. | **A7** | Low | `find_sync` `skip_samples` uses `.round()` where slowrx C truncates | `sync.rs:378` | slowrx `*Skip = s * Rate;` is implicit `double→int` (truncation toward zero). We `.round()`. Either match C (`as i64`) or document the `.round()` choice in `intentional-deviations.md` (rounding is arguably better — record the decision). | trivial |
| **A8** | Low | `find_sync` retry-exhaustion keeps last rate estimate vs slowrx C reset to 44100 — deviation undocumented | `sync.rs:302-315` | slowrx `sync.c:86-90` resets to nominal on retry exhaustion; we keep the estimate (reasonable — "re-anchoring a near-locked input is harmful"). | **A9** | Low | `SnrEstimator::estimate` can return `+∞` despite the documented `[-20 dB, …]` contract | `snr.rs:328-332` | Tiny-positive `p_noise` + large `p_signal` → `p_signal/p_noise == ∞`, `∞ < 0.01` false → returns `10·log10(∞) == +∞`.
